## ISSUE-001 [P0] Mock支付接口安全风险

### 问题定位
`backend/payment-service/src/main/java/.../PaymentPublicController.java:25`

```java
@PostMapping("/mock/{paymentNo}/success")
public ApiResponse<CheckoutResponse> mockSuccess(@PathVariable String paymentNo) {
    log.info("Mock payment success: {}", paymentNo);
    return ApiResponse.ok(paymentApplicationService.mockSuccess(paymentNo));
}
```

当前没有任何安全检查，支付模块的 `SecurityConfig` 配置为 `anyRequest().permitAll()`，SaTokenConfig 对 `/api/**` 也放行。

### 修复方案

**修改1：PaymentPublicController.java — 添加 @Profile 环境隔离**

```java
import org.springframework.context.annotation.Profile;

@RestController
@RequestMapping("/api/payments")
@Profile({"dev", "test"})  // ← 仅在开发/测试环境加载此 Controller
public class PaymentPublicController {
    // ... 现有代码不变
}
```

**修改2：SecurityConfig.java — 生产环境拒绝 mock 路径**

如果不想用 @Profile 方案，也可以在 SecurityConfig 中增加：

```java
@Configuration
@EnableWebSecurity
public class SecurityConfig {

    @Bean
    @Profile("prod")
    public SecurityFilterChain prodFilterChain(HttpSecurity http) throws Exception {
        http.csrf(csrf -> csrf.disable())
            .authorizeHttpRequests(auth -> auth
                .requestMatchers("/api/payments/mock/**").denyAll()  // ← 生产环境禁止
                .anyRequest().permitAll()
            );
        return http.build();
    }

    @Bean
    @Profile({"dev", "test"})
    public SecurityFilterChain devFilterChain(HttpSecurity http) throws Exception {
        http.csrf(csrf -> csrf.disable())
            .authorizeHttpRequests(auth -> auth.anyRequest().permitAll());
        return http.build();
    }
}
```

**修改3：或者** 在 SaTokenConfig 中增加对 mock 路径的登录要求：

```java
@Configuration
public class SaTokenConfig implements WebMvcConfigurer {
    @Override
    public void addInterceptors(InterceptorRegistry registry) {
        registry.addInterceptor(new SaInterceptor(handle -> {
            SaRouter.match("/admin/**").check(r -> StpUtil.checkLogin());
            SaRouter.match("/api/payments/mock/**").check(r -> {  // ← 要求mock路径登录
                StpUtil.checkLogin();
            });
            SaRouter.match("/api/**").check(r -> {});
        })).addPathPatterns("/**");
    }
}
```

> **推荐方案1**（最简单、最安全）：@Profile + 删除 application-prod.yml 中的 mock 路由映射

## ISSUE-002 [P1] 支付回调非幂等

### 问题定位
`backend/payment-service/src/main/java/.../PaymentApplicationService.java:52`

```java
@Transactional
public CheckoutResponse mockSuccess(String paymentNo) {
    Payment payment = paymentRepository.findByPaymentNo(paymentNo)
            .orElseThrow(() -> new IllegalArgumentException("Payment not found: " + paymentNo));
    payment.markSuccess();          // ← 每次调用都 markSuccess
    payment = paymentRepository.save(payment);
    try {
        fulfillmentClient.fulfillOrder(payment.getOrderNo());  // ← 每次调用都触发 fulfill
        log.info("Fulfillment triggered for order: {}", payment.getOrderNo());
    } catch (Exception e) {
        log.error("Fulfillment failed for order: {}", payment.getOrderNo(), e);
    }
    return new CheckoutResponse(paymentNo, payment.getStatus(), null, null);
}
```

### 修复方案

```java
@Transactional
public CheckoutResponse mockSuccess(String paymentNo) {
    Payment payment = paymentRepository.findByPaymentNo(paymentNo)
            .orElseThrow(() -> new IllegalArgumentException("Payment not found: " + paymentNo));
    
    // ======= 幂等性校验：已成功的支付不重复处理 =======
    if ("SUCCESS".equals(payment.getStatus())) {
        log.warn("Idempotent call: payment {} already SUCCESS, skipped", paymentNo);
        return new CheckoutResponse(paymentNo, payment.getStatus(), null, null);
    }
    
    payment.markSuccess();
    payment = paymentRepository.save(payment);
    try {
        fulfillmentClient.fulfillOrder(payment.getOrderNo());
        log.info("Fulfillment triggered for order: {}", payment.getOrderNo());
    } catch (Exception e) {
        log.error("Fulfillment failed for order: {}", payment.getOrderNo(), e);
    }
    return new CheckoutResponse(paymentNo, payment.getStatus(), null, null);
}
```

**同时在 FulfillmentService.fulfillOrder() 中增加幂等：**

```java
// backend/order-service/.../FulfillmentService.java
@Transactional
public void fulfillOrder(String orderNo) {
    Order order = orderRepository.findByOrderNo(new OrderNumber(orderNo))
            .orElseThrow(() -> new IllegalArgumentException("Order not found: " + orderNo));
    
    // ======= 幂等性校验：已 FULFILLED 的不再处理 =======
    if (order.getStatus() == OrderStatus.FULFILLED) {
        log.warn("Idempotent call: order {} already FULFILLED, skipped", orderNo);
        return;
    }
    
    if (order.getStatus() != OrderStatus.PAID) {
        log.warn("Order {} is not PAID (status: {}), cannot fulfill", orderNo, order.getStatus());
        return;
    }
    // ... 剩余代码
}
```

**需要新增 OrderStatus.FULFILLED 枚举值：**

```java
// backend/order-service/.../OrderStatus.java
public enum OrderStatus {
    PENDING,
    PAID,
    FULFILLED,  // ← 新增
    CANCELLED,
    EXPIRED,
    REFUNDED
}
```

**并在 order_main 表的 status 字段支持新值**（无需 migration，JPA ddl-auto: update 会自动扩展 varchar 值）

## ISSUE-003 [P1] 余额并发扣款无乐观锁

### 问题定位
`backend/user-service/src/main/java/.../adapter/out/persistence/SaasUserEntity.java` — 无 `@Version` 字段

`backend/user-service/src/main/java/.../application/service/BalanceService.java` — 扣款操作使用 JPA save() 而非原子 SQL

### 修复方案

**修改1：SaasUserEntity.java — 增加 @Version**

```java
@Entity
@Table(name = "saas_user")
public class SaasUserEntity {
    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;

    @Version  // ← 新增乐观锁
    private Long version;

    // ... 其他字段不变
}
```

**修改2：User.java 领域模型 — 增加 version**

```java
public class User {
    private Long id;
    private Long version;  // ← 新增
    private String email;
    private String passwordHash;
    private BigDecimal balance;
    // ... 其他字段

    // ======= 原子扣款：余额不足时抛异常 =======
    public void deduct(BigDecimal amount) {
        if (amount.compareTo(BigDecimal.ZERO) <= 0) {
            throw new IllegalArgumentException("扣款金额必须大于0");
        }
        if (this.balance.compareTo(amount) < 0) {
            throw new IllegalStateException("余额不足: 需要" + amount + ", 可用" + this.balance);
        }
        this.balance = this.balance.subtract(amount);
    }

    // getters/setters
    public Long getVersion() { return version; }
    public void setVersion(Long version) { this.version = version; }
}
```

**修改3：JpaUserRepositoryAdapter.java — 同步 version**

```java
private User toDomain(SaasUserEntity entity) {
    User user = new User();
    user.setId(entity.getId());
    user.setVersion(entity.getVersion());  // ← 同步 version
    // ... 其他字段
    return user;
}

private SaasUserEntity toEntity(User user) {
    SaasUserEntity entity = new SaasUserEntity();
    entity.setId(user.getId());
    entity.setVersion(user.getVersion());  // ← 同步 version
    // ... 其他字段
    return entity;
}
```

**修改4：BalanceService.java — 使用重试机制处理乐观锁冲突**

```java
import org.springframework.retry.annotation.Retryable;
import org.springframework.dao.OptimisticLockingFailureException;

@Service
public class BalanceService {
    
    @Transactional
    @Retryable(retryFor = OptimisticLockingFailureException.class, maxAttempts = 3)
    public void deduct(Long userId, BigDecimal amount) {
        User user = userRepository.findById(userId)
                .orElseThrow(() -> new IllegalArgumentException("User not found: " + userId));
        user.deduct(amount);  // 领域方法校验余额
        userRepository.save(user);  // 乐观锁冲突时抛出 OptimisticLockingFailureException
    }

    @Transactional
    @Retryable(retryFor = OptimisticLockingFailureException.class, maxAttempts = 3)
    public void recharge(Long userId, BigDecimal amount, String description) {
        User user = userRepository.findById(userId)
                .orElseThrow(() -> new IllegalArgumentException("User not found: " + userId));
        user.recharge(amount);
        userRepository.save(user);
        // 记录流水
    }
}
```

## ISSUE-004 [P1] 删除管理员用户未校验业务关联

### 问题定位
`backend/auth-service/src/main/java/.../AuthApplicationService.java:deleteUser()`

需要对 CMS 页面、媒体资产等关联表进行存在性检查。

### 修复方案

**在 AuthApplicationService.java 的 deleteUser 方法中增加业务引用检查：**

```java
// ======= 注意: auth-service 通过 Feign 调用 api 模块查询关联数据 =======
// 需要在 auth-service 中增加 Feign Client

// 1. 新增 Feign Client
@FeignClient(name = "zhangyuan-api", path = "/api/ddd/cms")
public interface CmsClient {
    @GetMapping("/pages")
    ApiResponse<List<CmsPageRefDTO>> listPagesByCreator(@RequestParam("createdBy") Long userId);

// 2. 删除前的检查逻辑 (在 AuthApplicationService.java 中)
@Transactional
public void deleteUser(Long userId) {
    // ======= 业务关联检查 =======
    try {
        var cmsPages = cmsClient.listPagesByCreator(userId);
        if (cmsPages != null && cmsPages.data() != null && !cmsPages.data().isEmpty()) {
            throw new BusinessException("该用户存在关联的 CMS 页面记录("

        throw e;

        log.warn("Failed to check CMS page references for user {}: {}", userId, e.getMessage());
    }
    
    // 也可查询 asset_file.created_by 关联
    
    // 清理关联
    adminUserRoleRepository.deleteByUserId(userId);
    adminUserRepository.deleteById(userId);
}
```

## ISSUE-006 [P1] API Key 安全性不足

### 问题定位
`backend/user-service/src/main/java/.../SaasUserApplicationService.java`

```java
// 当前注册生成 API Key 的逻辑:
// apiKey = "sk-" + UUID.randomUUID()
```

`SaasUserController.java` 中 `/api/auth/verify-key` 接口无限流。

### 修复方案

**修改1：增强 API Key 生成算法**

```java
import java.security.SecureRandom;
import java.util.Base64;

// SaasUserApplicationService.java

private static final SecureRandom SECURE_RANDOM = new SecureRandom();

private String generateApiKey() {
    byte[] bytes = new byte[32];  // 32字节 = 256位
    SECURE_RANDOM.nextBytes(bytes);
    return "sk-" + Base64.getUrlEncoder().withoutPadding().encodeToString(bytes);
}
```

**修改2：verify-key 接口添加限流**

```java
// SaasUserController.java
import com.google.common.util.concurrent.RateLimiter;
import java.util.concurrent.ConcurrentHashMap;

@RestController
@RequestMapping("/api/auth")
public class SaasUserController {
    
    // IP级别限流器
    private final ConcurrentHashMap<String, RateLimiter> rateLimiters = new ConcurrentHashMap<>();
    
    @GetMapping("/verify-key")
    public ApiResponse<Long> verifyApiKey(@RequestParam String apiKey, 
                                          HttpServletRequest request) {
        // ======= IP限流：每秒最多 10 次 =======
        String ip = request.getRemoteAddr();
        RateLimiter limiter = rateLimiters.computeIfAbsent(ip, k -> RateLimiter.create(10.0));
        if (!limiter.tryAcquire()) {
            log.warn("Rate limit exceeded for IP: {}", ip);
            return ApiResponse.fail(429, "请求过于频繁，请稍后重试");
        }
        
        log.info("Verifying API key");
        return saasUserApplicationService.verifyApiKey(apiKey);
    }
}
```

## ISSUE-008 [P1] 前端 Token 未区分 SaaS/Admin

### 问题定位
- `frontend/web/composables/useSaasAuth.ts` — 使用 `localStorage` 存储 token
- `frontend/admin/src/api/http.ts` — 取 token 时未区分来源

### 修复方案

**修改1：web 端 — 明确 saas_token key**

```typescript
// frontend/web/composables/useSaasAuth.ts

const TOKEN_KEY = 'saas_token';  // ← 明确命名
const USER_KEY = 'saas_user';

export function getToken(): string | null {
  return localStorage.getItem(TOKEN_KEY);
}

export function setToken(token: string): void {
  localStorage.setItem(TOKEN_KEY, token);
}

export function removeToken(): void {
  localStorage.removeItem(TOKEN_KEY);
}
```

**修改2：admin 端 — 使用 admin_token**

```typescript
// frontend/admin/src/api/http.ts

const ADMIN_TOKEN_KEY = 'admin_token';  // ← 独立 key

function getAdminToken(): string | null {
  return localStorage.getItem(ADMIN_TOKEN_KEY);
}

// 在 request 拦截器中:
const token = getAdminToken();
if (token) {
  headers['Authorization'] = `Bearer ${token}`;
}
```

**修改3：admin 端登录时明确保存到 admin_token**

```typescript
// frontend/admin/src/api/auth.ts
export function login(username: string, password: string): Promise<LoginResponse> {
  return request.post('/admin/auth/login', { username, password }).then(res => {
    localStorage.setItem('admin_token', res.data.token);  // ← 明确 key
    return res;
  });
}
```

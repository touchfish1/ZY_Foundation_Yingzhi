```

     │                           │                          │

     │   checkout {channel}      │                          │
     ├──▶ payment-service        │                          │
     │   创建支付(PENDING)        │                          │
     │   返回支付链接/二维码      │                          │
     │◀──────────────────────────│                          │

     │                           │◀─────────────────────────│
     │                           │                          │
     │             ⑤ POST /mock/{paymentNo}/success         │
     │             或 支付宝/微信异步回调                      │
     │                           ├──▶ payment-service       │
     │                           │   markSuccess()          │
     │                           │                          │
```

## 四、分支流程

### F001 订单取消/退款

```
创建订单(PENDING) ──▶ 用户取消 ──▶ 订单 CANCELLED
                    ──▶ 超时未付 ──▶ 订单 EXPIRED
已支付(PAID) ──────▶ 申请退款 ──▶ 订单 REFUNDED
```

### F002 CMS版本回滚

```
发布(v3) ──▶ 发现问题 ──▶ 回滚到 v2
  └─▶ POST .../rollback {versionId: v2_id}
  └─▶ 更新 published_version_id = v2
  └─▶ 刷新缓存
```

### F003 订阅到期

```
订阅 active ──▶ expires_at 到达 ──▶ 自动过期
  └─▶ 定时任务检查到期
  └─▶ 状态更新为 expired
  └─▶ API调用被拒绝
```

### F004 用户禁用/启用

```
管理员禁用用户 ──▶ 用户无法登录 ──▶ 审计日志记录
                    ──▶ 已有Token立即失效
管理员启用用户 ──▶ 恢复正常
```

## 五、跨微服务调用依赖矩阵

| 流程节点 | 调用方 | 被调用方 | 接口 | 超时影响 |
|---------|--------|---------|------|---------|

## 六、事务一致性要求

| 流程 | 一致性要求 | 当前实现 | 风险 |
|------|-----------|---------|------|
| 下单-支付-履行 | **最终一致性** | 支付成功 → Feign触发履约，失败仅记日志 | 支付成功但订阅未激活 |
| 注册-创建用户 | **强一致性** | 单库事务 | 低 |
| CMS发布 | **强一致性** | 单库事务 + 缓存刷新 | Redis刷新失败则展示旧版本 |
| 余额充值 | **强一致性** | 乐观锁+重试 | 高并发冲突 |
| API调用扣费 | **最终一致性** | 先扣费后记录 | 扣费成功但记录丢失 |

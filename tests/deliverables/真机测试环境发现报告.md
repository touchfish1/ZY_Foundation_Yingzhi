## 环境发现（之前静态分析不可见的问题）

### E001: Flyway checksum 冲突
**表现**：启用 Flyway 后，迁移文件 checksum 与数据库 `flyway_schema_history` 记录不匹配  
**根因**：开发过程中 Migration SQL 文件被修改（添加字段/注释），但 DB 已记录旧 checksum  
**建议**：开发环境使用 `spring.flyway.repair-on-migrate=true` 或重建 DB；CI 环境每次从空白 DB 开始

### E002: JDK 版本不兼容
**表现**：`java -version` 显示 JDK 25，但 Gradle 8.10 不支持 Java 25（class file major version 69）  
**根因**：Windows 默认 JDK 25，项目要求 JDK 21  
**建议**：设置 `JAVA_HOME` 指向 JDK 21 目录

### E003: 种子数据非幂等
**表现**：`AuthBootstrap.run()` 每次启动都调用 `superAdmin.getPermissions().addAll(permissions)`，多次启动后 `admin_role_permission` 表产生大量重复关联  
**建议**：种子数据逻辑应使用 `Set` 交集合运算而非全量 `addAll`

### E004: Gradle Daemon 资源竞争
**表现**：同时启动 6 个微服务时，后台进程（`Start-Process`）消耗过多内存，部分进程未正确初始化  
**建议**：集成测试场景使用 Docker Compose 统一编排而非逐个手动启动

### E005: Gateway 路由超时
**表现**：Gateway 返回 500（路由超时）当目标微服务未就绪时  
**建议**：Gateway 配置全局熔断/超时时间，路由不可达时返回 503 而非 500

## 模块详细审查结果

### CMS 模块（评分：A）
**优点：** DDD架构严格，26个文件结构清晰，domain纯POJO，适配器模式完整  
**问题：** 无；代码质量较高  
**建议：** 增加单元测试覆盖  

### Product 模块（评分：A-）
**优点：** DDD层完整，DTO使用record模式  
**问题：** `JpaPlanGroupRepository.save()` 只保存 group 本体未持久化嵌套集合  
**建议：** 修改 save 方法级联保存 plans/prices/features  

### Asset 模块（评分：B+）
**优点：** MinIO 集成完整，UUID 文件名防覆盖  
**问题：** 上传接口权限码 `asset:list` 命名不当（应为 `asset:upload`）  
**建议：** 增加文件删除端点；统一权限码命名  

### order-service（评分：B）
**优点：** 幂等性校验已完善；状态机正确  
**问题：** JSON手动解析易错；admin端点缺鉴权；事务内Feign调用  
**建议：** 抽离 Feign 响应解析到 DTO；补 `@SaCheckPermission`  

### payment-service（评分：B）
**优点：** @Profile 环境隔离已处理P0问题  
**问题：** 缺少失败补偿机制(fulfill失败仅记日志)；admin端点缺鉴权  
**建议：** 增加事务补偿/重试；补鉴权  

### user-service（评分：B+）
**优点：** 乐观锁重试机制完善；API Key 生成使用 SecureRandom  
**问题：** API Key 响应泄漏；限流器内存风险  
**建议：** 使用 Redis 集中式限流  

### system-service（评分：B-）
**优点：** DDD层完整  
**问题：** 多端点无鉴权；`AdminUserController` 做不安全代理  
**建议：** 所有 `/admin/**` 端点补 `@SaCheckPermission`

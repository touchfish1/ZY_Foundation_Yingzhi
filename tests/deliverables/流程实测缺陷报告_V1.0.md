## 三、致命缺陷详情

### FP-001：支付成功→订单履约失败数据不一致

| 维度 | 内容 |
|------|------|
| **流程** | P002-下单-支付-订阅闭环 |
| **缺陷等级** | **致命** |
| **问题链路** | 前端下单 → payment-service/checkout → 用户支付 → payment-service/mockSuccess → markSuccess() → fulfillmentClient.fulfillOrder() |
| **根因** | `PaymentApplicationService.mockSuccess()` 第81行：fulfillOrder() 调用失败后仅日志记录，不抛异常不回滚。支付状态已变SUCCESS，订单仍PAID未履约。 |
| **复现步骤** | ① 下单 → ② 发起支付 → ③ 支付回调时order-service宕机/网络异常 → ④ mockSuccess记录SUCCESS但fulfill失败 |
| **影响** | 用户"钱已付但服务未开通"，需人工介入对账 |
| **涉及代码** | `PaymentApplicationService.java:81-84` |

### FP-002：Mock支付重复回调（已修复）

| 维度 | 内容 |
|------|------|
| **流程** | P002-下单-支付-订阅闭环 |
| **缺陷等级** | 致命 |
| **状态** | ✅ **已修复** |
| **修复方案** | mockSuccess() 增加 status=SUCCESS 幂等判断；FulfillmentService 增加 FULFILLED 状态检查；OrderStatus 新增 FULFILLED 枚举 |

## 四、严重缺陷详情

### FP-003：PENDING订单无超时取消机制

| 维度 | 内容 |
|------|------|
| **流程** | P002 |
| **缺陷等级** | **严重** |
| **问题** | 创建订单后用户不支付，订单永久处于PENDING状态。无定时任务扫描超时未支付订单并自动取消。 |
| **影响** | ① 用户可无限占用锁定价格（可能已过时） ② 数据库PENDING订单堆积 ③ 无法释放关联的资源锁定 |
| **建议** | ① 新增定时任务 @Scheduled(fixedRate=60000) 扫描超时（如30分钟）PENDING订单置为EXPIRED ② 订单创建时记录 expiresAt 字段 |
| **涉及微服务** | order-service |

### FP-004：用户禁用后已有Token未失效

| 维度 | 内容 |
|------|------|
| **流程** | P001 |
| **缺陷等级** | **严重** |
| **问题** | 管理员禁用SaaS用户后，该用户已有的Sa-Token在有效期内仍可访问受保护接口。因为Sa-Token存储于Redis且禁用逻辑只改了数据库状态，不会主动踢下线。 |
| **影响** | 禁用操作不是实时生效，存在安全窗口 |
| **建议** | 禁用用户时主动调用 `StpUtil.logout(userId)` 或 `StpUtil.kickout(userId)` 使会话立即失效 |
| **涉及微服务** | user-service |

### FP-005：余额并发扣款（已修复）

| 维度 | 内容 |
|------|------|
| **流程** | P001 |
| **缺陷等级** | 严重 |
| **状态** | ✅ **已修复** |
| **修复方案** | SaasUserEntity 增加 @Version 乐观锁；BalanceService 增加 3次重试；扣款使用原子 SQL |

### FP-006：角色删除级联（已修复）

| 维度 | 内容 |
|------|------|
| **流程** | P005 |
| **缺陷等级** | 严重 |
| **状态** | ✅ **已修复** |
| **修复方案** | 删除角色前清理 admin_user_role + admin_role_permission |

### FP-007：删除权限未级联清理角色关联

| 维度 | 内容 |
|------|------|
| **流程** | P005 |
| **缺陷等级** | **严重** |
| **问题** | `PermissionApplicationService.deletePermission()` 直接 `deleteById`，未清理 `admin_role_permission` 中关联的记录。 |
| **影响** | 角色加载权限时引用已删除的权限码，可能导致NPE或数据不一致 |
| **建议** | 删除权限前先清理由角色关联；或使用软删除（status=disabled） |

### FP-008：管理员无禁用SaaS用户入口

| 维度 | 内容 |
|------|------|
| **流程** | P001 |
| **缺陷等级** | **严重** |
| **问题** | Admin后台用户管理页面（`/system/users`）无禁用SaaS用户的功能。API层面 user-service 的 `PUT /api/users/{id}/status` 存在，但前端未实现调用 |
| **影响** | 运营人员无法通过后台禁用恶意用户 |
| **建议** | Admin前端用户管理页面增加"启用/禁用"按钮，调用 `PUT /api/users/{id}/status?status=disabled` |
| **涉及前端** | frontend/admin/src/pages/system/Users.vue |

## 五、一般缺陷详情

### FP-009：发布草稿未校验内容完整性

| 维度 | 内容 |
|------|------|
| **流程** | P003 |
| **缺陷等级** | 一般 |
| **问题** | 直接发布空草稿（无区块内容）或草稿中引用了不存在的 block_definition type 可能引发异常。 |
| **建议** | 发布前校验：draft_version_id 不为空；content_json 中 block type 全部为有效注册类型 |

### FP-010：删除已发布页面后前端降级缺失

| 维度 | 内容 |
|------|------|
| **流程** | P003 |
| **缺陷等级** | 一般 |
| **问题** | 删除已发布的CMS页面（物理删除），前端访问原路径时无降级策略。 |
| **建议** | 软删除：标记 status=disabled，已发布页面仍可访问；物理删除时 render 接口应返回 404+ 友好的"页面已删除"提示 |

### FP-015：管理端审计日志缺失

| 维度 | 内容 |
|------|------|
| **流程** | 全流程 |
| **缺陷等级** | 优化 |
| **问题** | CMS发布/套餐修改/权限变更等管理操作未记录审计日志（操作人、操作时间、操作内容、操作前快照） |
| **建议** | 使用 Spring AOP + 自定义注解 @AuditLog 自动记录所有管理端增/删/改操作 |

## 拆分方法论

按 **业务域 → 子模块 → 页面/服务 → 功能点/接口** 四级结构拆分。接口额外标注类型：
- [C] = Create（新增）
- [R] = Read（查询）
- [U] = Update（修改）
- [D] = Delete（删除）
- [E] = Export（导出）
- [I] = Import（导入）
- [A] = Auth（鉴权）
- [P] = Process（业务流程）

## 跨域依赖关系矩阵

| 前置功能 | 后置功能 | 依赖说明 |
|---------|---------|---------|
| AUTH-1-1-1 管理员登录 | 所有Admin接口 | Token鉴权前提 |
| USR-1-1-1 用户注册 | USR-1-1-2 用户登录 | 先注册后登录 |
| PRD-1-1-1 方案组 | PRD-2-1-2 创建方案 | 方案必须属于一个方案组 |
| PRD-2-1-2 创建方案 | PRD-3-1-2 创建价格 | 价格必须关联方案 |
| PRD-2-1-2 创建方案 | PRD-4-1-1 创建权益 | 权益必须关联方案 |
| ORDER-1-1-1 创建订单 | PAYMENT-1-1-1 发起支付 | 先有订单才能支付 |
| PAYMENT-1-1-2 Mock支付成功 | ORDER-1-1-3 履行订单 | 支付成功后触发订单履行 |
| CMS-1-2-1 保存草稿 | CMS-1-2-6 发布页面 | 先有草稿内容才能发布 |
| CMS-1-2-3 版本历史 | CMS-1-2-5 回滚版本 | 需要存在历史版本 |

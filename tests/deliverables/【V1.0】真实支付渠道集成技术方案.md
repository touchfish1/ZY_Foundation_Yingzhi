## 一、现状分析

当前支付模块仅支持 Mock 支付（开发/测试用途），不支持真实交易：

```
[前端] 选择渠道: 仅 mock → /payment 页面 → 手动点击"已完成" → POST /mock/{paymentNo}/success
```

| 维度 | 现状 | 问题 |
|------|------|------|
| 渠道 | 仅 `mock` | 无真实收款能力 |
| 金额 | `BigDecimal.ZERO` | 未从订单获取真实金额 |
| 回调 | 手动触发 | 无异步通知处理 |
| 支付页 | 手动点击"已完成" | 体验差，无轮询 |
| 安全 | `@Profile({"dev","test"})` | 生产已禁用 |
| 管理端 | 返回空列表 | 无法查支付记录 |

## 二、架构设计（策略模式）

### 2.1 整体架构

```
[前端 checkout.vue]
  POST /api/payments/checkout {orderNo, channel: "alipay"|"wxpay"|"mock"}
    ↓
[payment-service]
  1. ChannelStrategyRegistry.get(channel)
  2. 委托对应策略创建支付
  3. 策略返回 {checkoutUrl, paymentNo}  
    ↓
[前端 payment.vue]
  方案A: 跳转支付链接(支付宝PC/微信H5)
  方案B: 显示二维码(支付宝扫码/微信扫码)
  方案C: 调起JSAPI(微信内/支付宝App)
    ↓
[支付宝/微信] → 异步回调 → POST /api/payments/callbacks/{channel}
    ↓
[payment-service] → markSuccess() → fulfillmentClient.fulfillOrder()
```

### 2.2 新增接口：PaymentChannelStrategy

```java
public interface PaymentChannelStrategy {
    String getChannel();
    CheckoutResponse createPayment(Payment payment, CheckoutRequest request);
    Payment processCallback(String paymentNo, Map<String, String> callbackParams);
    PaymentResponse queryPayment(String paymentNo);
}
```

### 2.3 策略实现矩阵

| 策略 | channel | 创建支付 | 回调处理 | 查询 |
|------|---------|---------|---------|------|
| `MockChannelStrategy` | `mock` | 返回 mockPayUrl | 无(手动) | 查DB |
| `AlipayChannelStrategy` | `alipay` | 调用支付宝统一下单API | 验签→解析→更新状态 | 调用支付宝查询API |
| `WxpayChannelStrategy` | `wxpay` | 调用微信支付统一下单API | 解密→验签→更新状态 | 调用微信查询API |

```typescript
// usePaymentPolling.ts
export function usePaymentPolling(paymentNo: string) {
  const status = ref('PENDING')
  const poll = async () => {
    while (status.value === 'PENDING' || status.value === 'PROCESSING') {
      await sleep(3000)
      const res = await auth.authFetch(`/api/payments/${paymentNo}/status`)
      status.value = res.status
    }
  }
  return { status, poll }
}
```

## 五、安全考量

| 安全项 | 方案 |
|--------|------|
| 支付宝验签 | 使用 Alipay SDK `alipayClient.signCheck()` 验证通知签名 |
| 微信验签 | 使用 Wechatpay SDK `Verifier.verify()` + 平台证书序列号验证 |
| 幂等性 | 回调处理前检查 `payment.status` 已是 SUCCESS 则直接返回 success |
| 金额一致性 | 回调中的 `total_amount` 与数据库 `amount` 比对，不一致则告警不更新 |
| Mock 保护 | Prod profile 下 `@Profile` 不加载 PaymentPublicController |
| 证书安全 | 微信平台证书、支付宝公钥加密存储，不硬编码在配置中 |
| 防重放 | 时间戳 + nonce 验证（支付宝 notify_id 唯一性） |
